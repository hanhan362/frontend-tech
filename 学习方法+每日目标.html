第一部分：学习方法（针对 408 + 数学一）

1. 数学：费曼学习法 + 三遍刷题法

核心原则：数学不是"看懂的"，是"算对的"。很多人上课听懂、考试不会，就是因为只看不算。你挂科大概率也是这个原因。
步骤做法时间分配
第一遍：上课听懂 课前花 10 分钟扫一眼下节课内容；课上记笔记（记思路不是抄板书）；下课立刻用 1 句话复述今天学了什么 课堂时间
第二遍：当天练习 当天晚上做 3-5 道课后习题，不看答案自己做，做不出来翻书，翻完书还是做不出再看答案 每天 40 分钟
第三遍：周复习 + 错题 周末把这周的错题重新做一遍，盖上答案从头写，能完整做出来才算过 每周末 2 小时
📌 费曼检验法：找一张白纸，假装你在给一个完全不懂的同学讲这道题。如果你能讲清楚每一步在做什么、为什么这么做，这道题你才真的会了。讲不清楚 = 没懂。

💡 方法：上课用英文理解概念（更透彻），课后用中文辅导书做对应章节的题。每学完一章，立刻做王道对应的选择题 + 大题。不要等到大三再碰 408，那时候已经晚了。

3. 英语：输入驱动法

你是中外合作，英语环境比普通学生好。利用这个优势：把日常娱乐变成英语输入。
  • 看 YouTube 而不是 B 站：订阅 CS 频道（Fireship、ThePrimeagen、Computerphile），每天刷 15 分钟
  • 读英文技术文档：别用中文翻译，强迫自己读 MDN、React 官方文档英文版
  • 背单词用 Anki：每天 20 个新词 + 复习旧词，用间隔重复记忆
  • 写作训练：每周用英文写一篇技术博客（200 字即可），发到 Dev.to

4. C / C++ 学习路线

📖 推荐教材：《C Primer Plus》（第 6 版）— Stephen Prata
零基础友好，指针和内存讲得最透。每天 1 章 + 3 道课后编程题。不要买谭浩强《C 程序设计》。
阶段语言教材重点何时
基础期 C 语言 《C Primer Plus》 变量→循环→数组→函数→指针→结构体→文件 大一暑假
巩固期 C + 数据结构 课程教材 用 C 实现链表、栈、树、图 大二上
进阶期 C++ 《C++ Primer》 类→继承→多态→STL（vector/map/sort/string） 大二下
应用期 C++ 刷题 LeetCode / 蓝桥杯 STL 熟练使用，算法实战 大三起
💡 C 和 C++ 的关系：C 是地基（指针、内存），C++ 是工具（STL 效率高）。408 考研用 C 足够，复试机试用 C++ 更好。
不要同时学，先 C 后 C++。把 C 的指针搞懂了，C++ 两个月就能上手。

线路做什么频率

算法线LeetCode / 洛谷刷题，从 Easy 开始，每天 1 题雷打不动每天 1 题

第四部分：具体执行技巧

🔑 番茄钟工作法

25 分钟专注 + 5 分钟休息 = 1 个番茄。每 4 个番茄休息 15 分钟。
最关键的是：25 分钟内手机静音、放抽屉、屏幕朝下。微信消息、短视频、游戏——所有这些都等到休息的 5 分钟再看。

🔑 图书馆 > 宿舍 > 床上

宿舍是堕落的温床。只要学习，去图书馆。到了图书馆就算不想学，周围人都在学你也会被带动。在宿舍里你有一万种方法分心。

🔑 每日复盘模板（睡前 5 分钟写）

✅ 今天完成了：_______________
❌ 今天没做到的：_______________
💡 明天改进一点：_______________

不是写作文，每行一句话。关键是诚实面对自己。

🔑 挂科课重修策略

重修不是"再听一遍课"。重修的正确姿势：
① 找出挂科的原因（没听懂？没做题？考前崩溃？）
② 找到这门课最好的网课资源（B 站 / 慕课），不要只依赖老师
③ 每周额外花 3 小时做这门课的题
④ 目标不是"60 分飘过"，是"85 分以上证明我进步了"

🔑 社交与学习平衡

找 1-2 个同样想考研的朋友组成学习小组。作用是：
• 互相监督打卡（微信群里每天发完成截图）
• 讨论难题（给别人讲题是最好的学习）
• 没有人能在孤独的自律中走很远，你需要同路人
最后一句：你挂过科，所以你没有"差不多就行"的资格。
每一天的每一个小时，要么在追赶，要么在落后。选择权在你。
